# 订单组合优化器

一个可直接部署到 GitHub Pages 的纯前端网页工具。

主要功能：

- 上传 Excel 文件，自动读取金额列
- 支持无表头 Excel，不会漏掉第一条金额
- 支持手动输入金额、拖拽上传、自定义目标金额
- 直接在页面展示分组结果，并支持导出 Excel
- 全部逻辑在浏览器本地执行，不需要后端服务

## 业务规则

- 单笔金额 `>= 目标金额` 的订单直接单独成组
- 单笔金额 `< 目标金额` 的订单参与组合
- 每个组合的总金额必须 `>= 目标金额`

优化目标优先级：

1. 满足条件的组合组数尽可能多
2. 每组总金额尽可能贴近目标金额
3. 未能成功分组的剩余金额尽可能大

## 本次改进

### 1. 修复无表头 Excel 导入

现在会先按二维表读取工作表，再判断首行到底是表头还是数据：

- 如果检测到 `amount` 或 `金额`，按表头列读取
- 如果首行本身就是数字，会把首行也当作真实金额处理

这样纯金额列表的 Excel 不会再漏掉第一条数据。

### 2. 升级组合算法

当前使用混合求解策略：

- 小规模订单：使用精确求解，优先保证分组数最优
- 大规模订单：使用子集和动态规划启发式，尽量贴近目标金额，同时保持性能

这比原来的简单贪心更符合“组数优先”的业务目标。  
例如反例 `1, 2, 87, 92, 97` 现在可以正确得到两组：

- `97 + 2 + 1 = 100`
- `92 + 87 = 179`

### 3. 去掉关键运行时 CDN 依赖

项目现在把 Excel 依赖直接放在本地：

- 本地脚本：`vendor/xlsx.full.min.js`

页面不再依赖外部 `xlsx` CDN，部署到 GitHub Pages 后更稳定。  
同时移除了 Google Fonts 依赖，改为系统字体栈。

## 本地打开

直接双击 [index.html](./index.html) 即可在浏览器中使用。

如果你希望本地起一个静态服务，也可以在项目目录执行：

```powershell
python -m http.server 8000
```

然后访问 `http://localhost:8000`

## Excel 格式要求

Excel 里不需要订单编号，只需要一列金额。

推荐表头：

- `amount`
- `金额`

如果没有表头，也支持直接上传单列纯金额表。

## 技术结构

- 页面：`index.html`
- 样式：`styles.css`
- 页面逻辑：`app.js`
- 求解器：`optimizer.js`
- Python 校验脚本：`order_group_optimizer.py`
- 本地 Excel 组件：`vendor/xlsx.full.min.js`

## 部署到 GitHub Pages

1. 把项目推送到 GitHub 仓库
2. 进入仓库 `Settings`
3. 打开 `Pages`
4. 选择 `Deploy from a branch`
5. 选择 `main` 和 `/ (root)`

发布成功后，访问：

`https://你的用户名.github.io/你的仓库名/`
